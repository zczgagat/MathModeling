# 动态规划

## 1. 定义

动态规划（Dynamic Programming，DP）将多阶段决策问题分解为若干子问题，并利用最优性原理：一个最优策略的后续决策必然构成相应子问题的最优策略。

它适合具有阶段、状态、决策和状态转移结构的问题，不是“随时间变化的规划”这一泛称。

## 2. 基本要素

- 阶段 \(t\)：决策发生的时点或步骤；
- 状态 \(s_t\)：描述当前决策所需的历史信息；
- 决策 \(a_t\)：当前可采取的动作；
- 转移：\(s_{t+1}=T_t(s_t,a_t)\)；
- 即时收益或成本：\(r_t(s_t,a_t)\)；
- 价值函数：从当前状态开始的最优累计收益。

## 3. Bellman方程

最大化问题的递推为：

$$
V_t(s)=\max_{a\in A_t(s)}\{r_t(s,a)+V_{t+1}(T_t(s,a))\}.
$$

终端条件为 \(V_{T+1}(s)=g(s)\)。最小化问题将 `max` 换为 `min`。

记录使价值函数达到最优的动作，即可从终端状态反向回溯出完整策略。

## 4. 典型例子：0-1背包

设前 \(i\) 件物品、容量为 \(c\) 时的最大价值为 \(V(i,c)\)。递推为：

$$
V(i,c)=
\begin{cases}
V(i-1,c),&w_i>c,\\
\max\{V(i-1,c),V(i-1,c-w_i)+v_i\},&w_i\le c.
\end{cases}
$$

状态规模为 \(O(nC)\)，需注意容量过大时会造成伪多项式复杂度。

## 5. 建模步骤

1. 判断问题能否拆成阶段和有限状态。
2. 定义足以描述未来影响的最小状态，避免状态爆炸。
3. 明确当前决策集合和可行条件。
4. 写出状态转移方程。
5. 写出边界或终端条件。
6. 确定自底向上表格法或自顶向下记忆化搜索。
7. 保存决策指针并回溯实际策略。
8. 与贪心、整数规划或小规模穷举结果核对。

## 6. 时间与空间复杂度

动态规划的关键是避免重复计算。若状态数为 \(|S|\)，每个状态平均有 \(|A|\) 个动作，复杂度通常为 \(O(T|S||A|)\)。

可以使用滚动数组降低空间，但如果需要恢复完整路径，必须保存决策或重新回溯。

## 7. Python实现框架

```python
dp = [[0] * (capacity + 1) for _ in range(n + 1)]
take = [[False] * (capacity + 1) for _ in range(n + 1)]

for i in range(1, n + 1):
    w, v = weights[i-1], values[i-1]
    for c in range(capacity + 1):
        dp[i][c] = dp[i-1][c]
        if w <= c and dp[i-1][c-w] + v > dp[i][c]:
            dp[i][c] = dp[i-1][c-w] + v
            take[i][c] = True
```

## 8. 常见应用

- 生产计划和设备更新；
- 库存补货；
- 资源分配；
- 路径和航段选择；
- 投资组合阶段决策；
- 维护、替换与能源调度。

## 9. 常见错误

- 状态没有包含影响未来决策的关键信息；
- 状态定义过大，导致计算爆炸；
- 忘记边界条件或不可行状态；
- 只输出最优值，不回溯实际方案；
- 把有全局耦合约束的问题简单套用 DP；
- 将贪心法当作动态规划。

## 10. 论文写法

应清晰解释阶段、状态、决策、转移和边界条件，并给出 Bellman 方程、复杂度和回溯方案。若状态离散化，应说明离散步长对精度的影响。
