# 多智能体仿真：NetLogo 与 Python Mesa

## 1. 定义

多智能体仿真（Agent-Based Modeling，ABM）把系统中的个体表示为具有属性、规则、目标和交互行为的智能体，通过个体层面的行为模拟群体层面的涌现现象。

它适合研究异质个体、局部交互、空间移动、适应性决策和政策干预，不等同于普通蒙特卡洛随机模拟。

## 2. 模型组成

- 智能体：居民、车辆、企业、动物等；
- 环境：网格、网络或连续空间；
- 状态：位置、资源、偏好、健康状态等；
- 行为规则：移动、交易、学习、传播；
- 交互规则：邻域接触、竞争、合作；
- 调度机制：同步、随机顺序或事件驱动；
- 宏观指标：拥堵、价格、感染规模、资源分布。

## 3. 建模步骤

1. 明确为什么个体差异和交互不可忽略。
2. 定义智能体类型、属性和状态。
3. 定义空间、网络和时间步长。
4. 写出每类智能体的行为规则。
5. 规定交互、更新顺序和边界条件。
6. 用现实数据初始化人口和环境。
7. 校准不易直接观测的行为参数。
8. 重复仿真并汇总概率分布。
9. 验证微观行为和宏观模式。
10. 进行情景、政策和敏感性分析。

## 4. ODD描述规范

ABM论文可使用 ODD 协议保证模型可复现：

- Overview：目的、实体、状态变量、时间尺度；
- Design concepts：涌现、适应、交互、随机性等；
- Details：初始化、输入数据和子模型。

建模手可先写一张行为规则表：

| 智能体 | 触发条件 | 行为 | 状态变化 |
|---|---|---|---|
| 居民 | 邻居感染 | 按概率感染 | 健康→感染 |
| 车辆 | 前方拥堵 | 减速或换道 | 速度/车道变化 |

## 5. NetLogo

NetLogo适合快速搭建网格或网络上的教学及中等规模 ABM。核心对象包括：

- turtles：移动智能体；
- patches：空间网格；
- links：智能体连接；
- observer：控制整体仿真。

优点是交互界面和可视化方便，缺点是与复杂数据管道、机器学习和大规模计算集成相对有限。

## 6. Python Mesa

Mesa 是 Python 的 ABM 框架，便于与 NumPy、Pandas、NetworkX 及统计工具结合。典型结构包括 Agent、Model、空间容器、激活机制和 DataCollector。

```python
from mesa import Agent, Model
from mesa.time import RandomActivation

class Person(Agent):
    def step(self):
        self.move()
        self.interact()

class Society(Model):
    def __init__(self, n):
        self.schedule = RandomActivation(self)
        for _ in range(n):
            self.schedule.add(Person(self.next_id(), self))

    def step(self):
        self.schedule.step()
```

不同 Mesa 版本的 API 可能变化，实际使用时应按安装版本调整。

## 7. 校准

设模拟宏观指标为 \(m_k(\theta)\)，真实指标为 \(y_k\)，可最小化：

$$
J(\theta)=\sum_kw_k[m_k(\theta)-y_k]^2.
$$

由于仿真具有随机性，每组参数应重复运行，再使用均值、分位数或分布距离作为校准目标。可采用网格搜索、贝叶斯优化或近似贝叶斯计算。

## 8. 验证与不确定性

- 规则验证：代码是否忠实实现概念模型；
- 极端情景：零交互、无限资源等情况下结果是否合理；
- 微观验证：个体行为分布是否接近数据；
- 宏观验证：群体模式是否再现现实；
- 样本外验证：在未参与校准的时期或地区测试；
- 随机性分析：多个种子下报告区间；
- 全局敏感性：识别驱动结果的关键参数。

## 9. 适用场景

- 疫情传播和人群行为；
- 交通、疏散和城市空间；
- 市场竞争、消费选择和舆论传播；
- 生态系统和资源竞争；
- 多主体能源管理和供应链。

如果只关心总体平均量，且个体差异不重要，微分方程或离散事件仿真可能更简洁。

## 10. 常见错误

- 行为规则凭空设定，没有数据或文献依据；
- 参数很多但数据不足，模型不可识别；
- 只展示动画，不给定量指标；
- 只运行一次随机仿真；
- 未说明更新顺序，导致结果不可复现；
- 用同一组数据完成校准和验证；
- 将能复现历史现象误认为证明了微观机制正确。

## 11. 论文写法

应提供 ODD 描述、智能体状态表、规则流程图、初始化、调度机制、参数来源和随机种子策略。结果需报告重复仿真分布、校准与验证、敏感性及政策情景，而不仅是可视化截图。
