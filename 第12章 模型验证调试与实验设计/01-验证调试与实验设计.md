# 模型验证、调试与实验设计

## 1. 三层正确性

1. 代码是否按设计执行；
2. 是否忠实实现数学公式与约束；
3. 结果是否符合现实机制。

程序不报错不等于结果正确。

## 2. 测试与断言

```python
assert len(weights) == X.shape[1]
assert np.isclose(weights.sum(), 1)
assert np.isfinite(X).all()
```

为标准化、距离、指标、目标函数和约束编写已知答案的小例子。

## 3. 基准比较

- 预测：均值、上一期、线性回归；
- 分类：多数类、逻辑回归；
- 优化：人工或贪心方案；
- 仿真：解析或简化模型。

复杂模型不能稳定超过基准时，应说明原因或选择简单模型。

## 4. 验证设计

独立样本用K折；不平衡分类用分层K折；重复主体按主体分组；时间序列滚动验证；空间数据空间分块。

## 5. 消融与灵敏度

逐项移除特征、约束和模块，证明改进有效。改变关键参数、数据处理和随机种子，检查核心结论是否稳定。

## 6. 调试流程

1. 构造最小复现实例；
2. 检查形状、范围和中间结果；
3. 与手算或另一实现对比；
4. 关闭随机和并行；
5. 一次只修改一个问题；
6. 修复后增加测试。

## 7. 实验记录

每次记录代码版本、数据版本、参数、种子、运行时间、指标、输出文件和结论。不同模型必须使用相同数据划分进行公平比较。
