# 聚类分析：K-means、DBSCAN 与层次聚类

## 1. 定义

聚类是一类无监督学习方法，在没有已知类别标签的情况下，根据样本之间的相似性将其划分为若干组，使组内样本尽量相似、组间样本尽量不同。

聚类结果不是天然存在的“真实类别”，而是由特征选择、距离度量、算法和参数共同决定的结构。

## 2. 数据准备

设数据矩阵为：

$$
X=(x_{ij})_{n\times p},
$$

其中 \(n\) 为样本数，\(p\) 为特征数。建模前通常需要：

1. 处理缺失值和异常值；
2. 删除无意义编号和泄漏信息；
3. 对量纲不同的连续变量标准化；
4. 根据研究目标选择特征；
5. 必要时用 PCA 降维；
6. 选择合适的距离或相似度。

欧氏距离适合连续变量；曼哈顿距离对部分异常偏差更稳健；分类变量可考虑 Hamming 距离或 Gower 距离。

## 3. K-means

### 3.1 原理

K-means 将样本分为 \(K\) 类，使组内平方和最小：

$$
\min_{C_1,\ldots,C_K}
\sum_{k=1}^{K}\sum_{x_i\in C_k}\|x_i-\mu_k\|_2^2,
$$

其中 \(\mu_k\) 为第 \(k\) 类的质心。

迭代步骤：

1. 初始化 \(K\) 个质心；
2. 将每个样本分到最近质心；
3. 用组内样本均值更新质心；
4. 重复至分组或目标函数稳定。

### 3.2 适用条件

- 特征主要为连续数值；
- 类簇近似球状、尺度相近；
- 需要预先指定簇数；
- 数据规模较大。

K-means 对异常值、初始点和特征尺度敏感。应使用 K-means++ 和多次随机初始化。

## 4. DBSCAN

DBSCAN 是基于密度的聚类方法，核心参数为：

- \(\varepsilon\)：邻域半径；
- `min_samples`：形成稠密区域所需的最少样本数。

若某点 \(\varepsilon\) 邻域内样本数达到阈值，则为核心点；与核心点密度可达的样本构成一类，其余可能被标为噪声。

优点：

- 不需要预先指定簇数；
- 能识别非球状簇；
- 可发现噪声点。

局限：

- 参数选择敏感；
- 不同密度的簇难以同时识别；
- 高维空间中距离区分度下降。

可用 k-distance 图辅助选择 \(\varepsilon\)，但仍需结合业务解释和稳定性验证。

## 5. 层次聚类

凝聚型层次聚类从每个样本单独成类开始，逐步合并距离最近的簇，最终形成树状图。常用链接方法：

- 单链接：两个簇最近样本间距离；
- 完全链接：最远样本间距离；
- 平均链接：所有跨簇样本距离的平均；
- Ward 法：选择使组内平方和增量最小的合并。

树状图可展示不同切割高度下的聚类结构。单链接容易产生“链化”，Ward 法通常适合标准化后的连续变量和欧氏距离。

## 6. 聚类数选择与评价

### 轮廓系数

设 \(a(i)\) 为样本与同簇样本的平均距离，\(b(i)\) 为与最近其他簇的平均距离：

$$
s(i)=\frac{b(i)-a(i)}{\max\{a(i),b(i)\}}.
$$

平均轮廓系数越接近1，通常分离效果越好。

其他方法包括：

- 肘部法：观察组内平方和随 \(K\) 变化的拐点；
- Calinski-Harabasz 指数：越大通常越好；
- Davies-Bouldin 指数：越小通常越好；
- 重采样稳定性：数据扰动后分组是否稳定。

指标只是辅助，最终簇数还应满足现实解释和使用目的。

## 7. Python实现

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans, DBSCAN, AgglomerativeClustering

kmeans = make_pipeline(
    StandardScaler(),
    KMeans(n_clusters=3, n_init=20, random_state=42)
)
labels_km = kmeans.fit_predict(X)

dbscan = make_pipeline(StandardScaler(), DBSCAN(eps=0.5, min_samples=5))
labels_db = dbscan.fit_predict(X)

hier = make_pipeline(
    StandardScaler(),
    AgglomerativeClustering(n_clusters=3, linkage="ward")
)
labels_hier = hier.fit_predict(X)
```

## 8. 建模步骤

1. 明确聚类对象和分群目的。
2. 选择具有区分意义的特征。
3. 预处理并选择距离度量。
4. 用多个算法和参数建立候选结果。
5. 使用内部指标和可视化评价。
6. 比较随机初始化或重采样下的稳定性。
7. 统计每一类的特征均值、规模和典型样本。
8. 根据现实含义为类别命名。

## 9. 常见错误

- 未标准化就用欧氏距离聚类；
- 将编号、总分及其组成指标同时加入，造成重复计权；
- 用肘部法强行寻找并不存在的明显拐点；
- 把 DBSCAN 的噪声类当成普通类别解释；
- 只画二维 PCA 图就断言高维聚类效果很好；
- 根据聚类结果倒推因果结论；
- 只运行一次 K-means，不检查初始化稳定性。

## 10. 论文写法

应说明特征选择、标准化、距离、参数与簇数选择依据，并报告内部指标、稳定性、各簇画像和代表样本。聚类名称应来自数据特征，不应预先设定后强行对应。
