# 支持向量机 SVM

## 1. 定义

支持向量机（Support Vector Machine，SVM）通过寻找具有最大间隔的分类超平面完成分类。距离决策边界最近、决定边界位置的样本称为支持向量。

对线性可分数据，希望找到：

$$
f(x)=w^Tx+b,
$$

使两类样本之间的几何间隔最大。

## 2. 软间隔SVM

现实数据通常不可完全线性分离，引入松弛变量 \(\xi_i\)：

$$
\min_{w,b,\xi}\quad
\frac12\|w\|^2+C\sum_{i=1}^{n}\xi_i
$$

$$
\text{s.t. }y_i(w^Tx_i+b)\ge1-\xi_i,
\qquad \xi_i\ge0.
$$

参数 \(C\) 控制间隔宽度与训练误差之间的权衡：

- \(C\) 大：更重视训练样本分类正确，可能过拟合；
- \(C\) 小：允许更多误差，边界更平滑。

## 3. 核函数

核技巧通过：

$$
K(x_i,x_j)=\phi(x_i)^T\phi(x_j)
$$

隐式映射到高维空间，而不直接计算 \(\phi(x)\)。

常用核函数：

### 线性核

$$
K(x,z)=x^Tz.
$$

适合特征很多、关系近似线性的任务。

### 多项式核

$$
K(x,z)=(\gamma x^Tz+r)^d.
$$

### RBF高斯核

$$
K(x,z)=\exp(-\gamma\|x-z\|^2).
$$

\(\gamma\) 大时单个样本影响范围较小，边界更复杂；\(\gamma\) 小时边界更平滑。

## 4. 支持向量回归 SVR

SVR 使用 \(\varepsilon\)-不敏感损失，希望预测误差在 \(\varepsilon\) 范围内不受惩罚：

$$
L_\varepsilon(y,f(x))=\max(0,|y-f(x)|-\varepsilon).
$$

它适合中小样本的非线性回归，但仍需选择 \(C\)、\(\gamma\) 和 \(\varepsilon\)。

## 5. 数据预处理

SVM依赖距离和内积，对特征尺度敏感，通常必须标准化。正确流程是：

1. 先划分训练集和测试集；
2. 仅用训练集拟合标准化参数；
3. 对训练集和测试集应用同一转换；
4. 把标准化和模型封装进 Pipeline。

类别特征需先编码。异常值过多时，应考虑稳健缩放、软间隔和异常值处理。

## 6. 建模步骤

1. 明确分类标签或回归目标。
2. 清洗数据并构造预测时可获得的特征。
3. 划分训练、验证和测试集。
4. 建立逻辑回归、线性回归等基准模型。
5. 使用 Pipeline 完成缩放和 SVM 训练。
6. 通过交叉验证搜索核、\(C\)、\(\gamma\) 等参数。
7. 在独立测试集上进行最终评价。
8. 分析支持向量、误分类样本和特征影响。

时间序列数据应按时间切分；分组数据应按主体划分，避免同一主体同时进入训练集和测试集。

## 7. Python实现

```python
from sklearn.pipeline import make_pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.svm import SVC
from sklearn.model_selection import GridSearchCV

pipe = make_pipeline(StandardScaler(), SVC())
params = {
    "svc__kernel": ["linear", "rbf"],
    "svc__C": [0.1, 1, 10, 100],
    "svc__gamma": ["scale", 0.01, 0.1, 1]
}
search = GridSearchCV(pipe, params, cv=5, scoring="f1_macro")
search.fit(X_train, y_train)
```

回归任务使用 `sklearn.svm.SVR`。数据量很大、特征稀疏时，可优先考虑 `LinearSVC` 或 `LinearSVR`。

## 8. 评价指标

分类任务可使用：

- 准确率；
- 精确率、召回率、F1；
- 混淆矩阵；
- ROC-AUC 或 PR-AUC；
- 类别不平衡时优先关注宏平均 F1、召回率和 PR-AUC。

SVR 可使用 MAE、RMSE、\(R^2\)。所有任务都应与简单基准比较，并报告交叉验证波动。

SVM原始输出不是概率。若需要概率，可使用校准方法；`probability=True` 会增加训练成本，也需检查概率校准质量。

## 9. 多分类与类别不平衡

SVM本质上是二分类器，多分类通常通过一对一或一对其余策略实现。类别不平衡时可：

- 设置 `class_weight="balanced"`；
- 调整分类阈值；
- 使用分层交叉验证；
- 对少数类召回率和误判代价单独分析。

重采样必须只在训练折内完成，不能在交叉验证前对全数据操作。

## 10. 解释与敏感性

线性 SVM 的系数可在标准化尺度上辅助解释；非线性核模型可使用置换重要性或 SHAP 等方法，但这些解释通常是近似的。还应检查：

- 参数 \(C,\gamma\) 改变后性能是否稳定；
- 支持向量比例是否过高；
- 删除某类特征后的性能变化；
- 不同随机划分或时间窗口下的结果。

## 11. 常见错误

- 未标准化特征；
- 用测试集反复调参；
- 类别严重不平衡时只报告准确率；
- 把 SVM 决策函数值直接解释为概率；
- 样本量巨大时仍使用 RBF 核，导致训练过慢；
- 网格搜索范围没有依据或过度搜索测试数据；
- 将相关性特征的重要性解释为因果效应。

## 12. 论文写法

应报告数据划分、特征处理、核函数、参数搜索范围、交叉验证方式和最终参数。结果至少应包含混淆矩阵及合适的分类指标，或 SVR 的误差指标，并与基准模型比较。
