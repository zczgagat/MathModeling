# 编程基础与环境配置

## 1. 编程手的职责

编程手负责把建模方案变为可靠程序：读取和清洗数据、实现公式与约束、运行实验、验证结果、生成论文图表，并保证队友能够复现。

## 2. 推荐技术栈

- Python：主语言；
- NumPy、Pandas：数值和表格；
- SciPy、statsmodels：数值计算和统计；
- scikit-learn：机器学习；
- Matplotlib、Seaborn：绘图；
- PuLP、OR-Tools、NetworkX：优化和图论；
- Jupyter：探索；Git：版本管理。

## 3. 必备Python能力

熟练掌握数据类型、条件循环、函数、异常、模块、文件路径、NumPy索引与广播、Pandas筛选分组和合并。

```python
from pathlib import Path
import pandas as pd

ROOT = Path(__file__).resolve().parent
data = pd.read_excel(ROOT / "data" / "input.xlsx")
```

不要把个人电脑绝对路径写入代码。

## 4. 环境配置

```powershell
python -m venv .venv
.\.venv\Scripts\activate
python -m pip install numpy pandas scipy matplotlib scikit-learn
python -m pip freeze > requirements.txt
```

比赛前应离线测试全部常用库和求解器。

## 5. 项目结构

```text
problem/
├─ data/raw/          # 原始数据，只读
├─ data/processed/    # 清洗数据
├─ src/               # 函数与模型
├─ notebooks/         # 探索分析
├─ outputs/figures/
├─ outputs/tables/
├─ main.py
└─ requirements.txt
```

## 6. 编码要求

- 变量名表达现实含义；
- 每个函数只做一件事；
- 参数集中配置；
- 检查形状、单位、缺失和范围；
- 固定随机种子并记录日志；
- 保留从原始数据到最终结果的一键入口。

## 7. 达标标准

应能在一小时内读取多表、检查缺失、合并字段、计算统计量、绘图、导出结果，并在队友电脑复现。
