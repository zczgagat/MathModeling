# 元胞自动机：交通流与火灾扩散

## 1. 定义

元胞自动机（Cellular Automaton，CA）将空间划分为规则网格，每个元胞具有有限状态，并按照局部邻域规则在离散时间同步或异步更新。宏观复杂行为由大量简单的局部交互产生。

一个元胞自动机通常由以下部分构成：

$$
\mathcal A=(L,S,N,F),
$$

- \(L\)：元胞空间；
- \(S\)：有限状态集合；
- \(N\)：邻域；
- \(F\)：局部状态转移规则。

## 2. 邻域与边界

二维常用邻域：

- Von Neumann 邻域：上、下、左、右四个相邻元胞；
- Moore 邻域：再加入四个对角元胞，共八邻域。

常见边界条件：

- 固定边界；
- 周期边界；
- 吸收或开放边界；
- 反射边界。

边界处理会显著影响传播和交通结果，必须根据现实场景选择。

## 3. 交通流模型

### 3.1 Nagel–Schreckenberg模型

一维道路划分为元胞，每个元胞最多容纳一辆车，车辆速度为整数：

$$
v_i\in\{0,1,\ldots,v_{\max}\}.
$$

每个时间步通常按以下规则同步更新：

1. 加速：\(v_i\leftarrow\min(v_i+1,v_{\max})\)；
2. 安全减速：\(v_i\leftarrow\min(v_i,g_i)\)，其中 \(g_i\) 是前方空元胞数；
3. 随机慢化：以概率 \(p\) 令 \(v_i\leftarrow\max(v_i-1,0)\)；
4. 移动：\(x_i\leftarrow x_i+v_i\)。

可分析密度 \(\rho\)、平均速度 \(\bar v\) 和流量：

$$
q=\rho\bar v.
$$

扩展模型可加入红绿灯、换道、匝道、不同车型和驾驶行为。

## 4. 火灾扩散模型

元胞状态可设为：

```text
0：不可燃区域
1：可燃但未燃烧
2：正在燃烧
3：已经燃尽
```

未燃元胞在邻域存在燃烧元胞时，以概率 \(P_{ignite}\) 被点燃。概率可受植被、湿度、坡度和风向影响：

$$
P_{ignite}=\operatorname{clip}
\left(P_0f_{fuel}f_{moisture}f_{slope}f_{wind},0,1\right).
$$

燃烧达到持续时间后转为燃尽状态。风向可以通过方向相关的邻域权重表达，使顺风方向的点燃概率更高。

## 5. 建模步骤

1. 明确研究区域、空间分辨率和时间步长。
2. 规定元胞状态及其现实含义。
3. 选择邻域与边界条件。
4. 根据机制、文献或数据设计转移规则。
5. 初始化道路车辆或空间可燃物分布。
6. 采用同步或异步方式迭代更新。
7. 记录宏观指标，如流量、拥堵长度、烧毁面积。
8. 用实际数据校准规则参数。
9. 多次随机仿真并统计均值、分位数和置信区间。

## 6. 同步更新

元胞自动机通常要求所有元胞根据时刻 \(t\) 的旧状态计算 \(t+1\) 状态。因此应使用两个网格：

```python
new_grid = grid.copy()
# 所有更新均读取 grid，写入 new_grid
grid = new_grid
```

若一边遍历一边直接修改原网格，结果会依赖遍历顺序，相当于使用异步更新。只有在模型明确要求异步机制时才应这样做。

## 7. Python实现框架

火灾扩散的简化示意：

```python
import numpy as np

EMPTY, TREE, BURNING, BURNT = 0, 1, 2, 3

def update(grid, ignite_probability, rng):
    rows, cols = grid.shape
    new_grid = grid.copy()
    for i in range(rows):
        for j in range(cols):
            if grid[i, j] == BURNING:
                new_grid[i, j] = BURNT
            elif grid[i, j] == TREE:
                neighbors = grid[max(0, i-1):i+2, max(0, j-1):j+2]
                if np.any(neighbors == BURNING):
                    if rng.random() < ignite_probability:
                        new_grid[i, j] = BURNING
    return new_grid
```

真实模型应单独处理风向、边界、燃烧持续时间及空间异质性。

## 8. 参数校准与验证

- 交通模型：利用流量—密度关系、速度分布、拥堵波传播速度校准；
- 火灾模型：利用历史火场边界、蔓延速度、烧毁面积和方向校准；
- 使用训练场景估计参数，在其他时段或事件上验证；
- 随机模型需重复运行，不能只挑选一次看起来合理的模拟；
- 改变网格尺寸和时间步，检查尺度敏感性；
- 与无风、低密度等已知极端情景比较。

## 9. 优缺点

优点：

- 能表达空间异质性和局部交互；
- 规则直观、便于可视化；
- 适合模拟非线性传播、拥堵和涌现行为；
- 易于加入障碍、地形和个体差异。

局限：

- 结果依赖网格、邻域和规则设计；
- 参数可能难以从数据中识别；
- 离散网格可能产生方向偏差；
- 随机模拟计算量较大且不直接给出解析结论。

## 10. 常见错误

- 转移规则没有现实或文献依据；
- 直接修改原网格，意外变成异步更新；
- 空间分辨率和时间步不匹配，使车辆或火势速度失真；
- 只展示动画或热力图，不报告量化指标；
- 随机模拟只运行一次；
- 训练和验证使用同一个事件；
- 网格结果漂亮，但未做参数和尺度敏感性分析。

## 11. 论文写法

应明确元胞状态、空间尺度、时间步、邻域、边界、更新顺序和随机规则。最好使用规则表或流程图展示演化机制，并报告校准数据、重复次数、宏观指标、置信区间及不同情景的比较结果。
