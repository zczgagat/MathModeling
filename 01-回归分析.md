# 回归分析（线性 / 多项式 / 岭回归）

> **一句话**：用已知变量预测未知变量，核心是找到一条"最佳拟合线"。

---

## 一、线性回归（Linear Regression）

### 数学原理

假设因变量 $y$ 与 $p$ 个自变量 $x_1, x_2, ..., x_p$ 呈线性关系：

$$y = \beta_0 + \beta_1 x_1 + \beta_2 x_2 + \cdots + \beta_p x_p + \varepsilon$$

用 **最小二乘法（OLS）** 估计参数：

$$\hat{\beta} = (X^TX)^{-1}X^Ty$$

目标：使残差平方和最小

$$RSS = \sum_{i=1}^{n}(y_i - \hat{y}_i)^2$$

### 适用场景

- 变量间关系近似线性
- 样本量 >> 变量数（建议 10:1 以上）
- 作为 baseline 模型，后续用复杂模型对比改进

### 建模步骤

```
① 散点图确认线性趋势
② 拟合模型
③ 显著性检验：F 检验（整体）+ t 检验（单个系数）
④ 模型诊断：
   - 残差正态性 → Q-Q 图
   - 异方差 → White 检验 / Breusch-Pagan 检验
   - 自相关 → Durbin-Watson 检验
   - 多重共线性 → VIF（方差膨胀因子）
⑤ 如果有多个变量 → 逐步回归筛选
```

### 模型评价指标

| 指标 | 含义 | 标准 |
|------|------|------|
| $R^2$ | 拟合优度（解释变异比例） | > 0.8 好；社科 > 0.4 可接受 |
| 调整 $R^2$ | 惩罚变量个数 | 越高越好 |
| AIC / BIC | 信息准则（模型选择） | 越低越好 |
| RMSE | 均方根误差（预测精度） | 越小越好 |
| MAE | 平均绝对误差 | 越小越好 |

### 在数学建模中的用法

- **几乎每题都可用作 baseline**，但很少作为最终模型（太简单）
- 与复杂模型做对比时套话："先用线性回归建立基准模型，再用 X 模型改进，RMSE 降低了 Y%"
- 注意分析残差图，发现非线性时自然过渡到多项式回归或非线性模型

---

## 二、多项式回归（Polynomial Regression）

### 数学原理

增加自变量的高次项以捕捉非线性关系：

$$y = \beta_0 + \beta_1 x + \beta_2 x^2 + \cdots + \beta_d x^d + \varepsilon$$

本质仍是线性模型（对参数 $\beta$ 线性），因此仍用 OLS 求解。

### 关键问题

| 问题 | 说明 | 解决方法 |
|------|------|---------|
| **次数选择** | d 过小欠拟合，d 过大过拟合 | 交叉验证选使 RMSE 最小的 d |
| **Runge 现象** | 高次多项式在边界剧烈震荡 | 使用样条插值替代 |
| **多重共线性** | $x$ 与 $x^2$ 高度相关 | 正交多项式 / 标准化后拟合 |

### 次数选择策略

```
d = 1 → 线性回归（baseline）
d = 2 → 捕捉一个拐点（开口向上/下）
d = 3 → 捕捉 S 形趋势
d ≥ 4 → 谨慎！需交叉验证验证泛化能力
```

### 建模注意事项

```python
# sklearn 实现
from sklearn.preprocessing import PolynomialFeatures
from sklearn.linear_model import LinearRegression
from sklearn.pipeline import Pipeline

model = Pipeline([
    ('poly', PolynomialFeatures(degree=3, include_bias=False)),
    ('linear', LinearRegression())
])
model.fit(X_train, y_train)
```

> **比赛中**：多项式回归是"你明明做了非线性，但写在论文里不会显得太难"的模型——适合用来给神经网络做对比基准。

## 三、岭回归（Ridge Regression）

### 为什么需要岭回归

OLS 在以下情况失效：

| 问题 | 现象 | 原因 |
|------|------|------|
| 多重共线性 | $X^TX$ 不可逆或接近奇异 | 变量间高度相关 |
| 过拟合 | 训练集 $R^2$ 很高，测试集很差 | 模型学到噪声 |
| 变量数 > 样本数 | OLS 无唯一解 | $p > n$ |

### 数学原理

在 OLS 损失函数中加入 **L2 正则化项**（也叫惩罚项）：

$$\min_{\beta} \left[ \sum_{i=1}^{n}(y_i - \hat{y}_i)^2 + \lambda \sum_{j=1}^{p} \beta_j^2 \right]$$

解为：

$$\hat{\beta} = (X^TX + \lambda I)^{-1}X^Ty$$

### 正则化参数 $\lambda$

| $\lambda$ | 效果 | 偏差-方差 |
|-----------|------|-----------|
| $\lambda = 0$ | = 普通线性回归 | 高方差 |
| $\lambda$ 较小 | 轻微收缩系数 | 降低方差 |
| $\lambda$ 较大 | 系数接近 0 | 高偏差（欠拟合）|
| $\lambda \to \infty$ | 所有 $\beta \to 0$ | 完全欠拟合 |

**选择方法**：**交叉验证**（RidgeCV 自动搜索最优 $\lambda$）

### 岭回归 vs 普通回归 vs Lasso

```
         普通回归           岭回归               Lasso
         ──────           ──────              ──────
正则化     无              L2 正则化            L1 正则化
特征筛选   不筛选           不筛选               自动筛选（系数=0）
系数特点   无偏但方差的      有偏但方差小         稀疏解
共线性     敏感             不敏感               不敏感
适用场景   数据干净          多重共线性           特征筛选
```

### 标准化的重要性

岭回归对变量尺度敏感！**必须先标准化**，否则大的变量的系数会被惩罚更多。

```python
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import RidgeCV

scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
model = RidgeCV(alphas=[0.01, 0.1, 1, 10, 100], cv=5)
model.fit(X_scaled, y)
print(f"最佳 lambda: {model.alpha_}")
```

## 四、三种回归的比赛使用建议

### 整体策略

```
① 拿到数据 → 先做线性回归（baseline）
② 检查残差 → 如果有非线性模式 → 多项式回归
③ 检查 VIF → 如果有共线性（VIF > 10）→ 岭回归
④ 特征太多 → Lasso（L1 正则化）
⑤ 最终在论文中呈现 baseline vs 改进的对比
```

### 论文写作模板

> 为建立 $y$ 与 $x$ 之间的预测关系，首先采用多元线性回归建立基准模型。经检验，模型存在多重共线性（VIF > 10），故引入岭回归加以改进。通过 5 折交叉验证确定最佳正则化参数 $\lambda = 0.5$，得到岭回归模型 $R^2 = 0.86$，相比线性回归的 $R^2 = 0.78$ 显著提升，且模型稳定性增强。

### 常见错误

- ❌ 不做模型诊断直接套用
- ❌ 不标准化就做岭回归
- ❌ 几个模型并列但不解释为什么选
- ❌ 只汇报 $R^2$ 不汇报 RMSE
- ✅ 正确做法：多个指标综合 + 残差分析 + 模型对比
