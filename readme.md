# 数学建模模型知识库

这是一个面向数学建模竞赛的中文模型速查与学习仓库，覆盖模型方法、编程实践与论文写作，共 **16 个专题、39 篇教程**。

每篇文档尽量按照统一结构编写：

- 模型定义与核心思想
- 数学公式与基本假设
- 适用场景和选择依据
- 完整建模步骤
- Python 实现框架
- 评价、检验与敏感性分析
- 常见错误
- 数学建模论文写法

> 模型越复杂不代表论文越优秀。比赛中应优先选择与问题机制、数据规模和评价目标相匹配的方法，并保证模型可解释、可求解、可验证。

## 二、按问题快速选择模型

| 题目特征 | 首选方法 | 可选改进 | 关键验证 |
|---|---|---|---|
| 多因素预测连续数值 | 线性/岭回归 | 多项式、BP、SVR | 测试集误差、残差 |
| 有时间顺序和季节性 | 指数平滑、ARIMA | Prophet、LSTM | 滚动验证、预测区间 |
| 小样本且近似单调趋势 | GM(1,1) | 回归趋势模型 | 级比检验、样本外误差 |
| 多指标方案排序 | TOPSIS | 熵权-TOPSIS、组合赋权 | 权重与排名敏感性 |
| 专家判断指标重要性 | AHP | AHP + TOPSIS | 一致性检验 |
| 评价等级边界模糊 | 模糊综合评价 | AHP/熵权赋权 | 隶属函数敏感性 |
| 指标过多且相关性强 | PCA | 因子分析 | 贡献率、载荷解释 |
| 成本或收益线性、资源有限 | 线性规划 | 整数/0-1规划 | 约束检查、对偶或敏感性 |
| 选址、分配、排程 | 整数/0-1规划 | 元启发式算法 | 精确小规模基准、最优间隙 |
| 路径和网络连接 | 最短路、生成树、TSP、最大流 | OR-Tools、GA/SA | 路径可行性、基准比较 |
| 多阶段序贯决策 | 动态规划 | 强化学习 | Bellman递推、策略对比 |
| 服务拥堵和库存补货 | 排队论、存储论 | 离散事件仿真 | 分布假设、重复仿真 |
| 连续时间机制和守恒关系 | 微分方程 | 参数反演、PDE | 守恒、拟合、敏感性 |
| 离散周期状态更新 | 差分方程 | 马尔可夫链 | 稳定性、多步误差 |
| 空间局部传播或涌现 | 元胞自动机 | 多智能体仿真 | 尺度、规则、重复实验 |
| 无标签样本分群 | K-means | DBSCAN、层次聚类 | 轮廓系数、稳定性 |
| 隐藏状态随时间变化 | HMM | 贝叶斯 HMM | 样本外似然、状态解释 |
| 中小样本非线性分类 | SVM | 树模型、神经网络 | F1、混淆矩阵、交叉验证 |
| 图像识别、计数或分割 | 图像处理 | CNN、迁移学习 | 独立测试、IoU/mAP |
| 评论、新闻和政策文本 | TF-IDF + 线性模型 | LDA、Transformer | 宏 F1、人工抽检 |
| 检验参数变化与结论稳定性 | 单因素扰动、情景分析 | Morris、Sobol、蒙特卡洛 | 排名、方案及输出稳定性 |

## 三、推荐学习路线

### 第一阶段：基础入门

建议先掌握最常用且容易解释的模型：

1. 数据清洗、描述统计和可视化；
2. 线性回归与误差评价；
3. TOPSIS、熵权法与 AHP；
4. 线性规划和整数规划；
5. 聚类分析；
6. 敏感性分析和模型验证。

### 第二阶段：按题型扩展

- 预测方向：时间序列、GM(1,1)、SVR；
- 优化方向：图论、动态规划、非线性规划；
- 机理方向：微分方程、差分方程；
- 数据方向：PCA、贝叶斯统计、马尔可夫模型。

### 第三阶段：综合与高阶方法

在已经能完成基础模型和论文验证后，再学习：

- 元启发式算法；
- 元胞自动机与多智能体仿真；
- 神经网络和强化学习；
- 计算机视觉与自然语言处理。

高阶方法应服务于题目需求，并与简单基准模型进行公平比较。

### 第四阶段：论文表达与团队实战

- 学会把每一问写成“目标—模型—结果—检验—结论”的证据链；
- 训练摘要、图表、公式、符号和参考文献的规范表达；
- 与建模手、编程手进行限时交接和整场模拟；
- 使用冻结结果表进行内容、数字和格式三轮审校。

## 四、比赛中的使用流程

拿到赛题后，不要先在目录中寻找“最高级模型”，建议按以下顺序工作：

1. **重述问题**：明确每一问要求的输入、输出和评价标准。
2. **拆分任务**：判断属于预测、评价、优化、机理还是数据分析。
3. **检查数据**：确认样本量、变量类型、时间与空间结构以及缺失情况。
4. **建立基准**：先完成一个简单、可解释、能运行的基础模型。
5. **选择改进**：根据基础模型缺陷增加约束、非线性或高阶算法。
6. **求解与检验**：进行误差、约束、残差、稳定性和敏感性分析。
7. **形成闭环**：确保模型输出确实回答题目，并能支持结论和建议。

可以为每个子问题填写以下建模卡片：

```text
问题目标：
输入数据：
输出结果：
决策变量或状态变量：
模型假设：
目标函数或核心方程：
约束条件：
求解方法：
评价指标：
验证方法：
主要局限：
```

## 五、模型组合示例

竞赛题通常需要多个模型形成完整链路，而不是孤立使用单一算法。

### 需求预测与资源调度

```text
时间序列/回归预测需求
        ↓
整数规划确定资源分配
        ↓
图论或TSP设计运输路线
        ↓
预测误差与容量敏感性分析
```

### 区域综合评价

```text
数据清洗与指标正向化
        ↓
PCA筛选或压缩指标
        ↓
AHP/熵权法确定权重
        ↓
TOPSIS计算综合排名
        ↓
权重扰动和排名稳定性分析
```

### 传播与干预决策

```text
SIR/元胞自动机描述传播
        ↓
历史数据估计参数
        ↓
优化模型选择干预方案
        ↓
多情景仿真与不确定性分析
```

## 六、论文质量检查

提交论文前至少确认：

- 每个变量、参数和单位均已定义；
- 假设与模型公式相互对应；
- 数据处理没有使用未来信息；
- 训练集、验证集和测试集划分合理；
- 优化结果满足全部现实约束；
- 随机算法进行了多次独立运行；
- 结果与简单基准或已有方法比较；
- 进行了误差、敏感性或稳健性分析；
- 图表具有标题、单位、图例和数据来源；
- 摘要给出了具体模型、关键结果和验证结论。

## 七、使用说明

本仓库中的公式、流程和代码均为学习与建模框架。实际参赛时需要根据题目重新定义变量、参数、约束和数据处理方式，不能直接套用模板。

建议每学完一个模型，都使用一份真实数据完成“数据处理—建模—求解—验证—论文表达”的完整练习，并把可复用代码单独整理成经过测试的工具库。
