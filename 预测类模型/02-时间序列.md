# 时间序列：ARIMA、指数平滑与 Prophet

## 1. 定义

时间序列预测利用观测值随时间变化的规律预测未来。它重点处理趋势、季节性和序列自身的相关性。

### 1.1 ARIMA

ARIMA 的记号为 ARIMA\((p,d,q)\)：

- \(p\)：自回归阶数；
- \(d\)：使序列平稳所需的差分次数；
- \(q\)：移动平均阶数。

对差分后的序列 \(z_t\)，模型可写为：

$$
z_t=c+\sum_{i=1}^{p}\phi_i z_{t-i}
+\varepsilon_t+\sum_{j=1}^{q}\theta_j\varepsilon_{t-j}.
$$

存在周期季节性时使用 SARIMA\((p,d,q)(P,D,Q)_s\)，其中 \(s\) 为季节周期。

### 1.2 指数平滑

指数平滑给予近期观测更高权重。简单指数平滑为：

$$
\ell_t=\alpha y_t+(1-\alpha)\ell_{t-1}.
$$

Holt 方法加入趋势，Holt-Winters 方法进一步加入季节项。季节波动幅度近似固定时用加法形式，随序列水平成比例变化时用乘法形式。

### 1.3 Prophet

Prophet 是一种可加分解模型：

$$
y(t)=g(t)+s(t)+h(t)+\varepsilon_t,
$$

其中 \(g(t)\) 为分段趋势，\(s(t)\) 为周期性，\(h(t)\) 为节假日或事件效应。它适合包含多重季节性、节假日和部分缺失值的业务序列。

## 2. 适用场景

- ARIMA/SARIMA：单变量、等间隔、具有明显自相关的序列。
- 指数平滑：水平、趋势和季节结构较清晰，要求模型简单稳健。
- Prophet：日级或更细业务数据，存在周、年季节性及节假日效应。

典型问题包括客流、销量、负荷、需求量、气象指标和设备监测值预测。

## 3. 建模前检查

1. 时间是否连续且间隔一致；
2. 是否存在缺失时点、异常事件或统计口径变化；
3. 是否有趋势和季节性；
4. 预测步长是多少；
5. 数据长度是否覆盖多个完整周期；
6. 是否存在可提前获得的外生变量。

ARIMA 一般要求差分后的序列平稳，可用时序图、ACF/PACF 以及 ADF 检验辅助判断。

## 4. 建模步骤

1. 按时间排序并统一采样频率。
2. 对缺失值和异常值作有依据的处理。
3. 保留最后一段数据作为测试集。
4. 建立朴素预测或季节朴素预测作为基准。
5. 根据趋势、季节和自相关选择候选模型。
6. 在训练集上估计参数，使用滚动验证选择配置。
7. 检验残差是否接近白噪声。
8. 生成点预测和预测区间。

## 5. 三类方法的使用要点

### ARIMA

- 通过差分消除趋势，避免过度差分。
- ACF/PACF 可辅助选择 \(p,q\)，AIC/BIC 可用于候选模型比较。
- 残差应无显著自相关，可使用 Ljung-Box 检验。
- 若有外生变量，可使用 ARIMAX 或 SARIMAX。

### 指数平滑

- 无趋势、无季节：简单指数平滑。
- 有趋势：Holt 双指数平滑。
- 有趋势和季节：Holt-Winters。
- 对长期预测，要谨慎判断趋势是否会无限延续，可使用阻尼趋势。

### Prophet

- 数据通常整理成 `ds`（时间）和 `y`（数值）两列。
- 根据题目加入节假日、特殊事件和额外回归变量。
- 调节趋势变化点和季节性的灵活程度，避免把噪声当规律。
- Prophet 使用方便，但不意味着一定优于传统方法，必须与基准比较。

## 6. 评价方法

可使用 MAE、RMSE、MAPE 或 sMAPE。真实值可能为零时，MAPE 会失真，应改用 MAE、RMSE 或 sMAPE。

时间序列推荐滚动验证：每次只用过去预测未来，再逐步向前移动窗口。这更接近真实使用场景。

除点预测误差外，还应评价预测区间覆盖率。例如标称 95% 的区间应覆盖约 95% 的真实值。

## 7. Python工具

- ARIMA/SARIMA：`statsmodels.tsa.arima.model.ARIMA`、`SARIMAX`
- 指数平滑：`statsmodels.tsa.holtwinters.ExponentialSmoothing`
- Prophet：`prophet.Prophet`

```python
from statsmodels.tsa.holtwinters import ExponentialSmoothing

model = ExponentialSmoothing(
    train,
    trend="add",
    seasonal="add",
    seasonal_periods=12
).fit()
forecast = model.forecast(steps=6)
```

## 8. 常见错误

- 随机划分训练集和测试集，泄漏未来信息。
- 数据不足一个完整季节周期，却强行建立季节模型。
- 只追求最低 AIC，不检查样本外误差和残差。
- 对非平稳序列直接使用普通 ARMA。
- 将缺失日期直接删除，破坏等间隔结构。
- Prophet 参数全部使用默认值，且不与朴素预测比较。
- 预测跨度远大于有效历史长度，却不给出不确定性说明。

## 9. 比赛中的选择顺序

先画时序图并建立朴素基准；结构清晰时优先尝试指数平滑，明显自相关时尝试 ARIMA/SARIMA，存在节假日和多重季节性时再考虑 Prophet。最终以滚动验证误差、可解释性和题目需求共同决定，而不是依据模型名的新旧决定。
