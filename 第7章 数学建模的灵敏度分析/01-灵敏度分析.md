# 数学建模的灵敏度分析

## 1. 定义

灵敏度分析研究模型输入、参数、假设或数据发生变化时，模型输出会发生多大变化。它主要回答：

- 哪些参数最影响结果？
- 参数在合理范围内变化时，核心结论是否稳定？
- 排名、最优方案或预测趋势会不会改变？
- 应优先收集或提高哪些数据的精度？

设模型输出为：

$$
y=f(x_1,x_2,\ldots,x_p),
$$

灵敏度分析就是研究每个输入 \(x_i\) 及其交互作用对 \(y\) 的影响。

灵敏度分析不同于不确定性分析：前者强调“哪些输入影响最大”，后者强调“输入不确定性会使输出产生怎样的分布”。两者通常结合使用。

## 2. 为什么数学建模论文需要灵敏度分析

模型中的权重、增长率、成本、容量、阈值和初始条件往往并非绝对准确。如果参数稍有变化，结论就完全相反，模型的实际可信度就很低。

灵敏度分析可以：

1. 检验结论的稳健性；
2. 识别关键参数；
3. 发现模型结构缺陷；
4. 支持政策和资源配置建议；
5. 为后续数据收集确定优先级；
6. 说明模型的适用范围与风险。

## 3. 基本流程

1. 明确需要观察的输出指标；
2. 选择待分析的参数或输入变量；
3. 确定参数基准值、合理范围和概率分布；
4. 选择局部或全局灵敏度方法；
5. 设计参数扰动或抽样方案；
6. 重复运行模型并记录输出；
7. 计算灵敏度指标并绘图；
8. 判断核心结论是否稳定；
9. 给出管理建议与模型局限。

参数范围应来源于历史波动、测量误差、置信区间、文献或专家意见，不能随意统一设为 \(\pm10\%\)。

## 4. 单因素扰动法

单因素扰动法又称 OAT（One-at-a-Time）：每次只改变一个参数，其他参数保持基准值不变。

设参数基准值为 \(x_i^0\)，可取：

$$
x_i=x_i^0(1+\delta),
\qquad
\delta\in\{-20\%,-10\%,0,10\%,20\%\}.
$$

然后观察输出：

$$
y(\delta)=f(x_1^0,\ldots,x_i^0(1+\delta),\ldots,x_p^0).
$$

优点是简单直观，适合比赛快速分析；缺点是不能充分识别参数之间的交互作用，也可能遗漏非线性区域。

## 5. 局部灵敏度分析

### 5.1 偏导数

参数 \(x_i\) 的局部绝对灵敏度为：

$$
S_i=\frac{\partial y}{\partial x_i}.
$$

数值模型可使用中心差分近似：

$$
S_i\approx
\frac{f(x_i+h)-f(x_i-h)}{2h}.
$$

### 5.2 归一化灵敏度

不同参数量纲不同时，可使用弹性系数：

$$
E_i=\frac{\partial y}{\partial x_i}\frac{x_i}{y}.
$$

它表示参数相对变化 1% 时，输出大约变化多少百分比。例如 \(E_i=0.8\)，表示在基准点附近，参数增加 1%，输出约增加 0.8%。

局部方法仅反映基准点附近的变化。对于强非线性、阈值或多峰模型，应使用全局方法。

## 6. 情景分析

情景分析同时改变一组具有现实关联的参数，构造不同未来情景。例如：

| 情景 | 需求增长 | 资源容量 | 单位成本 |
|---|---:|---:|---:|
| 乐观 | +5% | +15% | -5% |
| 基准 | 0 | 0 | 0 |
| 悲观 | +20% | -10% | +15% |

情景不应只是参数的机械排列，而应具有现实解释。可以比较各情景下的预测值、最优方案、成本、风险和约束违反情况。

## 7. 蒙特卡洛灵敏度与不确定性分析

若输入参数具有概率分布，可重复抽样并运行模型：

$$
x^{(k)}\sim p(x),\qquad
y^{(k)}=f(x^{(k)}),quad k=1,\ldots,N.
$$

输出可报告：

- 均值、中位数和标准差；
- 5%、50%、95% 分位数；
- 置信区间或预测区间；
- 超过风险阈值的概率；
- 输入与输出的相关系数或秩相关系数。

参数相关时不能独立抽样，应使用相关矩阵、Copula 或联合分布保留依赖关系。

## 8. Morris筛选法

Morris方法适合参数较多、单次模型运行成本较高的情况。它通过在参数空间中计算多条轨迹上的基本效应，得到：

- \(\mu^*\)：平均绝对基本效应，越大表示总体影响越强；
- \(\sigma\)：基本效应标准差，越大表示非线性或交互作用越明显。

Morris法主要用于筛选重要参数，而不是精确分解输出方差。筛选后可对少数关键参数进行 Sobol 分析。

## 9. Sobol全局灵敏度分析

Sobol方法将输出方差分解为各参数及交互作用贡献。若：

$$
Y=f(X_1,\ldots,X_p),
$$

一阶 Sobol 指数为：

$$
S_i=\frac{\operatorname{Var}_{X_i}
\left(E[Y\mid X_i]\right)}{\operatorname{Var}(Y)}.
$$

它表示参数 \(X_i\) 单独解释的输出方差比例。

总效应指数为：

$$
S_{T_i}=1-
\frac{\operatorname{Var}_{X_{\sim i}}
\left(E[Y\mid X_{\sim i}]\right)}{\operatorname{Var}(Y)}.
$$

它包含参数 \(X_i\) 的主效应及所有交互作用。若 \(S_{T_i}-S_i\) 很大，说明该参数与其他参数存在明显交互作用。

Sobol分析需要大量模型运行，参数范围和分布必须有依据。

## 10. 常用图表

### 灵敏度曲线

横轴为参数变化比例，纵轴为模型输出，适合展示单因素非线性变化。

### 龙卷风图

比较每个参数从低值到高值时输出变化的范围。条形越长，参数越敏感。

### 热力图

同时改变两个参数，展示输出或方案变化，适合发现交互和阈值区域。

### 箱线图或小提琴图

展示不同情景或随机抽样下的输出分布。

### 排名稳定图

展示评价权重扰动后，各方案名次变化或进入前 \(k\) 名的概率。

## 11. 不同模型中的灵敏度分析

### 11.1 预测模型

可以改变：

- 训练时间窗口；
- 特征选择；
- 模型超参数；
- 缺失值和异常值处理方式；
- 输入变量的测量误差。

观察 MAE、RMSE、预测趋势和区间覆盖率是否稳定。时间序列应使用滚动验证，不要只扰动训练集后报告拟合误差。

### 11.2 综合评价模型

重点扰动指标权重：

$$
w_i'=w_i(1+\delta_i),
$$

扰动后应重新归一化：

$$
\tilde w_i=\frac{w_i'}{\sum_jw_j'}.
$$

观察 TOPSIS 得分、方案排序和 Spearman 秩相关系数。若小幅权重变化就导致第一名频繁改变，应报告方案排名不稳定。

### 11.3 优化模型

可以改变：

- 成本和收益系数；
- 资源容量；
- 市场需求；
- 约束上限；
- 风险偏好参数。

观察最优目标值、决策变量、约束松弛量和方案是否改变。线性规划还可利用影子价格和允许增减范围分析参数变化。

### 11.4 微分方程与机理模型

可以改变传播率、恢复率、增长率、扩散系数和初始条件，观察峰值、峰值时刻、稳态和累计量。

例如 SIR 模型中：

$$
R_0=\frac{\beta}{\gamma}.
$$

应分别扰动 \(\beta\) 与 \(\gamma\)，分析感染峰值及其出现时间，而不应只分析 \(R_0\) 一个组合量。

### 11.5 随机仿真与元启发式算法

需要区分参数变化与算法随机性。每组参数均应使用多个随机种子运行，并报告输出均值、标准差和置信区间，避免把随机波动误判为参数效应。

## 12. Python实现：单因素扰动

```python
import numpy as np
import pandas as pd

def one_at_a_time(model, baseline, ranges):
    records = []
    for name, changes in ranges.items():
        for change in changes:
            params = baseline.copy()
            params[name] = baseline[name] * (1 + change)
            output = model(**params)
            records.append({
                "parameter": name,
                "change": change,
                "output": output,
            })
    return pd.DataFrame(records)

baseline = {"demand": 100, "capacity": 120, "cost": 8}
ranges = {
    name: np.array([-0.2, -0.1, 0, 0.1, 0.2])
    for name in baseline
}
results = one_at_a_time(model, baseline, ranges)
```

如果参数基准值为零，应使用绝对扰动而不是相对扰动。

## 13. Python实现：蒙特卡洛分析

```python
import numpy as np

rng = np.random.default_rng(42)
n = 10000

demand = rng.normal(100, 10, n)
cost = rng.triangular(6, 8, 12, n)
outputs = np.array([
    model(demand=d, cost=c)
    for d, c in zip(demand, cost)
])

summary = {
    "mean": outputs.mean(),
    "std": outputs.std(ddof=1),
    "q05": np.quantile(outputs, 0.05),
    "median": np.quantile(outputs, 0.50),
    "q95": np.quantile(outputs, 0.95),
}
```

全局灵敏度可使用 `SALib` 实现 Morris 和 Sobol 方法。抽样和分析函数必须匹配，且参数边界需符合真实含义。

## 14. 如何判断模型是否稳健

模型稳健性不能只用一句“变化不大”描述，应预先设置判据，例如：

- 参数变化 \(\pm10\%\)，核心输出变化不超过 5%；
- 排名第一的方案在 90% 的扰动情景下保持第一；
- 最优方案在所有合理情景下均满足约束；
- 预测方向和关键拐点没有改变；
- 主要管理建议在不同参数组合下保持一致。

判据应根据问题风险和业务要求制定，而不是所有题目使用同一阈值。

## 15. 常见错误

- 所有参数都机械扰动 \(\pm10\%\)，没有现实依据；
- 一次同时改变多个参数，却把结果归因于某一个参数；
- 只分析目标函数值，不检查最优方案是否切换；
- 评价权重扰动后忘记重新归一化；
- 随机算法每组参数只运行一次；
- 只展示图表，不说明结论是否稳定；
- 将相关性系数直接等同于因果灵敏度；
- Sobol抽样数量不足，却过度解释细小指数差异；
- 只挑选支持结论的参数范围。

## 16. 比赛中的最低可用方案

时间紧张时，至少完成：

1. 选择 3—5 个关键参数；
2. 根据数据或文献设置合理变化范围；
3. 使用单因素扰动或三种情景；
4. 记录核心输出、排名或最优方案；
5. 绘制灵敏度曲线或龙卷风图；
6. 明确指出最敏感参数和稳定区间；
7. 给出对应的管理建议。

若模型包含明显交互、阈值或非线性，再增加二维热力图、蒙特卡洛、Morris 或 Sobol 分析。

## 17. 论文写法模板

可按以下逻辑撰写：

> 为检验模型结论对参数不确定性的稳健性，选取参数 \(a,b,c\) 进行灵敏度分析。参数范围根据历史数据的波动区间及相关文献确定。在保持其他参数为基准值的条件下，分别改变单个参数并重新求解模型。结果表明，输出对参数 \(a\) 最敏感；当 \(a\) 在给定范围内变化时，核心方案保持不变，而参数 \(b\) 超过临界值后会导致方案切换。因此，模型在常规情景下具有较好稳健性，但在高 \(b\) 情景下应采用备用方案。

论文中应同时提供参数范围表、灵敏度指标、图表和现实解释。结论不稳定并不意味着分析失败；准确指出临界参数和风险区间本身就是有价值的结果。
