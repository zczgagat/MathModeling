# 协作、版本管理与结果交付

## 1. 团队协作

团队必须共享同一套变量定义、数据版本、模型参数、图表编号和最终结果。编程手应交付可直接用于论文的材料，而非零散截图。

## 2. Git最低用法

```powershell
git status
git add src/main.py
git commit -m "实现需求预测基准模型"
git log --oneline
```

按功能小步提交，不提交虚拟环境、缓存和敏感数据。

## 3. 文件命名

```text
figure_01_demand_trend.png
table_03_model_comparison.xlsx
result_route_baseline.csv
```

不要使用“最终版2、真的最终版”等名称。

## 4. 配置和日志

参数放入YAML/JSON配置。日志记录数据规模、参数、随机种子、求解状态、运行时间和主要指标。每次运行保存实际配置。

## 5. 一键运行

主程序应能通过 `python main.py` 自动完成读取、清洗、建模、评价、绘图和导出。耗时流程可提供快速和完整模式。

## 6. 交付包

每个问题交付：方法说明、结果表、关键图、指标、基准比较、灵敏度结论、失败情景、文件路径和生成命令。

## 7. 备份与合规

原始数据至少两份，代码用Git管理，论文和输出定期归档。遵守比赛网络、软件和AI规定，记录外部数据来源，不公开隐私数据或API密钥。
