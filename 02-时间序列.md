# 时间序列分析（ARIMA / 指数平滑 / Prophet）

> **一句话**：按时间顺序排列的数据点，利用过去推断未来。

---

## 一、指数平滑（Exponential Smoothing）

### 核心思想

> 近期数据权重大，远期数据权重小，权重按指数衰减。

### 三种形式

| 形式 | 公式 | 适用场景 |
|------|------|---------|
| **一次平滑**（SES） | $S_t = \alpha y_t + (1-\alpha)S_{t-1}$ | 无趋势、无季节 |
| **二次平滑**（Holt） | 水平 + 趋势 | 有趋势、无季节 |
| **三次平滑**（Holt-Winters） | 水平 + 趋势 + 季节 | 有趋势、有季节 |

### Holt-Winters 完整模型（加法）

$$\begin{aligned}
L_t &= \alpha(y_t - S_{t-m}) + (1-\alpha)(L_{t-1} + T_{t-1}) \\
T_t &= \beta(L_t - L_{t-1}) + (1-\beta)T_{t-1} \\
S_t &= \gamma(y_t - L_t) + (1-\gamma)S_{t-m} \\
\hat{y}_{t+h} &= L_t + hT_t + S_{t-m+\text{mod}(h,m)}
\end{aligned}$$

### 平滑系数

| 符号 | 含义 | 范围 | 作用 |
|------|------|------|------|
| $\alpha$ | 水平平滑系数 | [0,1] | 越大对新数据反应越快 |
| $\beta$ | 趋势平滑系数 | [0,1] | 控制趋势更新速度 |
| $\gamma$ | 季节平滑系数 | [0,1] | 控制季节成分更新 |

所有系数通过**最小化 SSE（平方和误差）** 自动确定。

### 优缺点

| ✅ 优点 | ❌ 缺点 |
|---------|---------|
| 计算极快、实现简单 | 只捕捉线性趋势和固定周期 |
| 可解释性强 | 对突变不敏感 |
| 适合短期预测 | 长期预测效果差 |
| 数据量需求小（十几个点即可） | 不能外推复杂模式 |

### 比赛建议

- 数据少（< 30 个点）且趋势明显时使用
- 作为时间序列预测的 baseline
- 将 Holt-Winters 结果分解为水平/趋势/季节三部分画图，论文加分

## 二、ARIMA 模型

### 模型结构

ARIMA$(p, d, q)$ 由三部分组成：

| 部分 | 含义 | 参数 | 作用 |
|------|------|------|------|
| **AR**（自回归） | 用过去值预测当前值 | $p$ | 捕捉自相关 |
| **I**（差分） | 使序列平稳 | $d$ | 消除趋势/季节 |
| **MA**（移动平均） | 用过去的误差修正预测 | $q$ | 捕捉冲击效应 |

### 数学形式

$$(1-\phi_1B - \cdots - \phi_p B^p)(1-B)^d y_t = (1+\theta_1B + \cdots + \theta_q B^q)\varepsilon_t$$

其中 $B$ 是滞后算子：$B y_t = y_{t-1}$

### Box-Jenkins 建模步骤

```
Step 1: 平稳性检验
        └→ ADF 检验，p < 0.05 则平稳
           不平稳 → 差分（d=1 或 d=2）
           
Step 2: 识别 p, q
        └→ 看 ACF（自相关函数）和 PACF（偏自相关函数）
        
Step 3: 模型拟合
        └→ 最大似然估计 MLE
        
Step 4: 残差检验
        └→ Ljung-Box Q 检验：残差应为白噪声
           （p > 0.05 说明信息已充分提取）
           
Step 5: 模型选择
        └→ 在候选模型中选 AIC / BIC 最小的
        
Step 6: 预测
        └→ 输出点预测 + 置信区间
```

### ACF / PACF 识别口诀

| 模型 | ACF | PACF |
|------|-----|------|
| AR$(p)$ | 拖尾（指数衰减或正弦衰减） | $p$ 阶后截尾 |
| MA$(q)$ | $q$ 阶后截尾 | 拖尾 |
| ARMA$(p,q)$ | 拖尾 | 拖尾 |
| ARIMA$(p,d,q)$ | 差分后拖尾 | 差分后拖尾 |

### 季节 ARIMA — SARIMA

$$SARIMA(p,d,q)(P,D,Q)_m$$

- $(p,d,q)$：非季节部分
- $(P,D,Q)$：季节部分
- $m$：周期长度（如 $m=12$ 为月度数据年周期）

**常用 $m$ 值**：
- 月度数据：$m=12$
- 季度数据：$m=4$
- 周数据（工作日）：$m=5$
- 日数据：$m=7$

### 优缺点

| ✅ 优点 | ❌ 缺点 |
|---------|---------|
| 理论基础扎实 | 数据要求 > 50 个点 |
| 给出预测置信区间 | 对缺失值/异常值敏感 |
| 中短期预测效果好 | 本质线性模型 |
| 成熟的软件支持 | 参数选择有一定经验门槛 |

### 比赛建议

```python
# 快速自动定阶（避免手动看 ACF/PACF）
import pmdarima as pm

model = pm.auto_arima(
    y, seasonal=True, m=12,
    start_p=0, max_p=5, start_q=0, max_q=5,
    information_criterion='aic',
    stepwise=True, trace=True
)
print(model.summary())
```

> 比赛中时间有限时，用 `auto_arima` 自动选参，然后重点分析残差和预测结果。但论文中要写出最终选定的 $(p,d,q)$ 和理由。

## 三、Facebook Prophet

### 背景

2017 年 Facebook 开源，专门为**业务时间序列预测**设计，核心思想是将时间序列分解为可解释的成分。

### 数学模型

$$y(t) = g(t) + s(t) + h(t) + \varepsilon_t$$

| 成分 | 含义 | 数学形式 |
|------|------|---------|
| $g(t)$ | 趋势项 | 分段线性 / 逻辑增长曲线 |
| $s(t)$ | 季节项 | Fourier 级数（年/周/日） |
| $h(t)$ | 节假日效应 | 用户自定义的哑变量 |
| $\varepsilon_t$ | 误差 | 正态分布 |

### 三个核心设计

**① 趋势 $g(t)$** — 自动检测变点（changepoint）

```
分段线性趋势：g(t) = (k + a(t)δ) · t + (m + a(t)γ)
              ↑ 基准增长率  ↑ 变点处的调整
```

Prophet 自动在数据中检测趋势拐点，用户无需手动标注。

**② 季节 $s(t)$** — Fourier 级数逼近

$$s(t) = \sum_{n=1}^{N} \left[a_n \sin\left(\frac{2\pi nt}{P}\right) + b_n \cos\left(\frac{2\pi nt}{P}\right)\right]$$

- $P$：周期（年=365.25，周=7）
- $N$：Fourier 阶数（年 $N=10$，周 $N=3$）

**③ 节假日 $h(t)$** — 用户灵活自定义

可指定任意日期作为节假日，每个节假日前后可加"窗口期"。

### 使用示例

```python
from prophet import Prophet
import pandas as pd

# 准备数据：两列 ds（日期）, y（数值）
df = pd.DataFrame({'ds': dates, 'y': values})

model = Prophet(
    yearly_seasonality=True,
    weekly_seasonality=True,
    daily_seasonality=False,
    changepoint_prior_scale=0.05,  # 趋势灵活度（默认 0.05）
    seasonality_prior_scale=10.0,  # 季节灵活度
)
model.fit(df)

future = model.make_future_dataframe(periods=30)
forecast = model.predict(future)

# 可视化
fig = model.plot(forecast)           # 预测图
fig2 = model.plot_components(forecast)  # 分解图（趋势+季节+节假日）
```

### 关键参数

| 参数 | 含义 | 默认值 | 调参方向 |
|------|------|--------|---------|
| `changepoint_prior_scale` | 趋势灵活度 | 0.05 | 过拟合↓，欠拟合↑ |
| `seasonality_prior_scale` | 季节影响强度 | 10.0 | 季节波动弱↓，强↑ |
| `changepoint_range` | 变点窗口范围 | 0.8 | 早期数据没变点则减小 |
| `seasonality_mode` | 季节模式 | 'additive' | 季节幅度随趋势增长则换'multiplicative' |

### 优缺点

| ✅ 优点 | ❌ 缺点 |
|---------|---------|
| 自动处理缺失值和异常值 | 理论基础不如 ARIMA 严谨 |
| 几行代码完成建模+画图 | 对跳跃突变可能滞后 |
| 可解释性强（分解图） | 长期预测不确定性大 |
| 不需要平稳化处理 | 本质仍是加法分解模型 |

### 三种时间序列模型的选择

```
数据 < 30 个点       → 指数平滑（简单稳定）
数据 30-100 个点     → ARIMA / 指数平滑
数据 > 100 个点      → Prophet / ARIMA
有明确节假日效应     → Prophet（独有优势）
需要置信区间         → ARIMA 或 Prophet
只需短期预测         → 指数平滑（最快速）
需要解释趋势变化     → Prophet（分解图一目了然）
```

## 四、模型评价方法

### 时间序列特有的交叉验证

```python
# 滚动预测验证（Time Series Split）
from sklearn.model_selection import TimeSeriesSplit
tscv = TimeSeriesSplit(n_splits=5)
for train_idx, test_idx in tscv.split(X):
    # train: 前 N 个时间点
    # test:  第 N+1 个时间点（或接下来几个）
    ...
```

> 和普通 CV 的区别：**不能打乱时间顺序**，只能用过去预测未来。

### 预测误差指标

| 指标 | 公式 | 特点 |
|------|------|------|
| **MAE** | $\frac{1}{n}\sum\|y_t - \hat{y}_t\|$ | 直观，无单位敏感 |
| **RMSE** | $\sqrt{\frac{1}{n}\sum(y_t - \hat{y}_t)^2}$ | 对大误差惩罚更大 |
| **MAPE** | $\frac{100\%}{n}\sum\frac{\|y_t - \hat{y}_t\|}{\|y_t\|}$ | 百分比，可跨序列比较 |
| **SMAPE** | $\frac{200\%}{n}\sum\frac{\|y_t-\hat{y}_t\|}{\|y_t\|+\|\hat{y}_t\|}$ | MAPE 改进版，不受 0 除影响 |

## 快速参考卡

```python
# ========== 指数平滑（statsmodels） ==========
from statsmodels.tsa.holtwinters import ExponentialSmoothing
model = ExponentialSmoothing(
    y, trend='add', seasonal='add', seasonal_periods=12
).fit()
pred = model.forecast(12)

# ========== ARIMA（pmdarima 自动定阶） ==========
import pmdarima as pm
model = pm.auto_arima(y, seasonal=True, m=12, trace=True)
pred, conf_int = model.predict(n_periods=12, return_conf_int=True)

# ========== Prophet ==========
from prophet import Prophet
model = Prophet(yearly_seasonality=True, weekly_seasonality=True)
model.fit(df)
future = model.make_future_dataframe(periods=30)
forecast = model.predict(future)

# ========== 平稳性检验 ==========
from statsmodels.tsa.stattools import adfuller
p_value = adfuller(y)[1]  # p < 0.05 则平稳

# ========== ACF/PACF 图 ==========
from statsmodels.graphics.tsaplots import plot_acf, plot_pacf
plot_acf(y.diff().dropna())
plot_pacf(y.diff().dropna())
```
